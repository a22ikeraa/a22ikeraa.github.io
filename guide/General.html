# Introducción aos compoñentes

# Que é un compoñente?

Un compoñente é unha peza de software reutilizable. Con esto referímonos a que é un programa que vai ser reutilizado noutros proxectos.

En terminos de java podese decir que un compoñente é unha clase creada para ser reutilizada.

Estes están definidos polo duas funcions principais:

  • Estado: O estado son un conxunto de propiedades, as cales poden ser modificadas para adaptar o compoñente ao programa no que se insira.

  • Comportamento: Definido por os eventos aos que responde e os métodos que executa ante ditos eventos.

Programar un compoñente desde cero é unha ardua tarea, pois seria necesario porgramar desde como se renderiza en pantalla ata como responde aos eventos.

Pero esto non é necesario para o noso caso pois o que se vai facer é coller como base un compoñente de JSwing e estendelo.

# Java Bean

Un JavaBean é un compoñente software que se pode reautilizar e que pode ser manipulado visualemente por unha ferramenta de programación en linguaxe Java. Para isto ten unha interface que permite ao IDE interrogar o compoñente e coñecer as propiedades que define e os tipos de eventos que pode xerar en resposta a diversas accións.

Características dun JavaBean:

  • Persistencia: Debe poder gardase o estado do Bean enha vez fou perosnalizado polo programador.

  • Introspección: Debe permitir ao IDE poder recoñecer certos elementos de deseño como os nomes das porpiedades, xunto co tipo de datos e os métodos.

  • Personalización: A aparencia e conducta do Bean pode ser alterada pola persoa programadora mediante os cambios dos valires nas propiedades

  • Eventos: Debe informar ao IDE dos sucesos que pode xerar en resposta ás accións das persoas usuarias e támen os eventos que pode manexar.

Estas características son obligatorias para a creación de un compoñente.
